AI Agents App Review Brief

Static, JavaScript-free review page for external reviewers such as Claude. The live app is a React SPA, so this page exists to provide readable context without executing client-side JavaScript.

Live app: https://ai-agents-31pz.onrender.com/

GitHub repo: https://github.com/rudrapratapsinghrathore3609/Kuhu.github.io

Current goal: Build a shareable multi-agent workspace where agents can answer questions, learn reusable patterns from prompts, use connectors/sources, keep chat history, and help build or improve websites/apps.

Product Summary

AI Agents is a multi-agent chat workspace. It lets a user choose specialized agents, upload files/photos, maintain conversation history, save learned memory, use AI provider accounts, inspect source trails, and ask a coder agent for app-building help. It is intended to feel closer to ChatGPT than a raw dashboard while keeping power-user tools available in a drawer or tools panel.

Primary Agents

  • Jarvis: Chief Supervisor and router for broad tasks.
  • Nova: News and general knowledge agent.
  • Phil: Market research and R&D.
  • Mastermind: Finance and stock market.
  • Homelander: Skill learning and tracking.
  • Noir: Studies and academics.
  • Kuhu: Website planning/building agent with website tooling knowledge.
  • Coder: App builder and code upgrade agent.
  • Automate: Task automation agent.
  • History: Historical context and explanations.

Core Features

  • Supabase authentication for sign in and account creation.
  • Supabase-backed chat history, messages, uploads, memories, connectors, and AI accounts.
  • Shared AI accounts via Render environment variables, especially Gemini for free/shared online use.
  • Personal AI accounts saved per user inside the app.
  • Local Ollama support through OpenAI-compatible endpoint: http://localhost:11434/v1.
  • File/photo upload with text extraction for text files.
  • Source trail after answers, including active model, memory considered, and connector status.
  • Voice command controls and speak-answer button.
  • Agent collaboration/council concept for multi-agent results.
  • Coder agent for planning GitHub issues, app improvements, and implementation guidance.

Current Architecture

  • Frontend: React + Vite.
  • Backend: Express server running with tsx src/server/index.ts.
  • Database/Auth: Supabase.
  • Deployment: Render free web service.
  • Repository: GitHub repo rudrapratapsinghrathore3609/Kuhu.github.io.
  • AI providers: OpenAI-compatible API, Gemini, Anthropic/Claude, and local Ollama compatible mode.

Important Implementation Notes

  • The React app requires JavaScript, so simple fetchers only see an empty HTML shell. This page is static to solve that review problem.
  • Render free tier is too small for hosting meaningful local LLMs. Online shared use should use Gemini or another hosted provider.
  • Ollama works locally on the owner laptop but cannot be used by deployed Render unless exposed through a protected proxy/tunnel/VPN.
  • Raw Ollama should not be exposed publicly without authentication, rate limiting, HTTPS, and request filtering.
  • Shared AI provider accounts should be configured as Render environment variables, not typed by every user.

Known Problems / Recent Fixes

  • Users previously saw Invalid API key during sign-up because Supabase public/server keys were misconfigured.
  • Users previously saw only Jarvis or missing agents because server routes depended too much on a failing service-role path.
  • Server now returns built-in fallback agents and uses the signed-in user session for normal data routes.
  • AI account sharing is now designed around environment variables such as SHARED_GEMINI_API_KEY.
  • Raw server errors have been improved, but provider failures still need better UI recovery.
