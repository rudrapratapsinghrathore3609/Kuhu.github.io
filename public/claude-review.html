Current Architecture

Frontend: React + Vite, plus small static UI enhancement scripts.
Backend: Express + tsx on Render.
Auth: Supabase Auth, bearer token required for /api routes.
Database: Supabase tables for accounts, conversations, messages, memories, uploads, connectors, usage.
AI providers: Shared env accounts plus user-saved accounts. Gemini, Groq, OpenRouter, Together, OpenAI, Ollama-compatible.
Safety: Coder UI confirmation gate, quota guard, provider health cooldown, source trail after responses.

Known Risks To Review

Risk

Shared provider abuse

Quota guard exists, but please review whether daily request/token limits are strong enough for public sharing with friends.

Risk

Coder safety is UI-first

The Coder confirmation gate currently prevents risky prompts in the browser UI, but the backend should eventually enforce action permissions if real code execution tools are added.

Risk

Source quality is partly self-reported

The app appends a Source Trail, and the UI displays source cards, but web search connectors still need deeper live retrieval and source quality scoring.

Review

Multiple enhancement scripts

The UI has several static enhancement scripts. They are throttled now, but please review whether any should be consolidated into React components later.
